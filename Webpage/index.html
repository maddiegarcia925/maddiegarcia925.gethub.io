Introduction

Our product is a model that allows the user to input information on the GPU's benchmarks to predict it's thermal design power. The GPU benchmarks allow for easy comparison between multiple graphics cards by scoring their performance on a standardized series of tests. This can be useful in many instances, such as buying or building a new PC.

Our target value will be the thermal design power (TDP). The TDP is an indicator of the maximum amount of heat generated by a computer chip. It's essentially a reading that helps determine the power efficiency and performance of a component.

To create our model, we used thousands of PerformanceTest benchmark results. This data set is updated daily at PassMark.com with new graphics card benchmarks. With PerformanceTest V9, the GPU Compute score includes an OpenCL GPU test in its calculation, such as the KNN (k-nearest neighbor) model, the SVM (support vector machines) model.

Why EDA

Exploratory Data Analysis (EDA) uses existing data to determine the accuracy of a predictive model by observing patterns and any possible relationships in the data. Additionally, it identifies the best model for a given situation.

Size and Features

After cleaning the dataset so that it would be usable in our machine learning models, it has 7 columns and 410 rows. The 7 features are: G3Dmark, G2Dmark, price, gpuValue, test date, and category.

Target Variable

TDP (Thermal design power) refers to the maximum amount of heat a computer chip, including a CPU or GPU, is expected to generate under general usage.

Data Preprocessing Steps

Remove Null Values

There were too many missing values in four of the dataset's columns as seen in the graph. To prevent problems with the machine learning, all the rows containing NA values were removed.

Irrelevant Columns

The "gpuName" column was dropped because it contained too many unique values and it did not help in predicting the TDP value. Some of the GPU names are stated below in the word cloud. The “powerPerformance” column was also dropped since it required the TDP data to be calculated.

Feature Encoding

The "category" column’s string features were converted into numerical values so that it could be used in the machine learning.

Data Splitting

The dataframe’ s columns were split into x and y. The TDP column would be the y value and the rest would be the x value. The data was also split into test and train data. The train data is 80% of the original data and is fed into the machine learning model to discover any patterns. The other 20%, the test data, is used the evaluate the model and test its accuracy.
